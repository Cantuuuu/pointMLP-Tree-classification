2.5 Evaluación

Cada modelo se evalúa en 4 escenarios: su propio test set y el test set del otro dominio. Se utiliza Test Time Augmentation (TTA) con 10 augmentaciones ligeras (rotación Z aleatoria + jitter leve), promediando las probabilidades softmax antes de tomar argmax.

4. Discusión

4.1 ¿Los datos sintéticos son un sustituto viable para datos reales?

No en su estado actual. La caída de 68.8 puntos porcentuales al evaluar Model A en datos reales demuestra que los datos sintéticos generados procedimentalmente no capturan las características esenciales del LiDAR real. El modelo aprende a distinguir las "versiones sintéticas" de cada especie, pero estas representaciones no corresponden a la variabilidad geométrica real.

4.2 Magnitud del domain gap

El domain gap es asimétrico:

  • Sintético→Real: −68.8pp (catastrophic)
  • Real→Sintético: −9.1pp (significativa, pero el modelo real parte de un baseline más bajo)

Esto sugiere que el dominio sintético es un subconjunto simplificado del espacio de formas arbóreas, mientras que el dominio real contiene mucha más variabilidad no capturada por el generador.

4.3 ¿Para qué clases funciona mejor la transferencia?

Ninguna clase transfiere bien en la configuración actual. PINE muestra algo de consistencia en recall para A→B (el modelo detecta "algo" de PINE en reales), pero con demasiados falsos positivos. La categoría OTHER es inherentemente difícil por ser una clase "catchall".

4.4 Limitaciones

  • Dataset pequeño: ~770 muestras por dominio, ~600 de entrenamiento, limita la capacidad del modelo.
  • Solo coordenadas XYZ: No se utilizan features adicionales como intensidad de retorno, número de retornos, RGB, o índices espectrales.
  • 3 clases agrupadas: La categoría OTHER agrupa especies muy diferentes, diluyendo patrones aprendibles.
  • Generación sintética simplificada: Los árboles procedimentales pueden no capturar ramificación real, densidad de copa, o patrones de oclusión del sensor.
  • Sin domain adaptation: No se aplican técnicas de adaptación de dominio que podrían reducir el gap (adversarial training, MMD, etc.).

4.5 Posibles mejoras

  • Domain adaptation: Técnicas como DANN o MMD loss para alinear representaciones entre dominios.
  • Mejores sintéticos: Incorporar ruido realista, oclusiones, y variabilidad morfológica basada en datos botánicos.
  • Más datos reales: Aumentar el dataset real a >2000 muestras.
  • Features adicionales: Incorporar intensidad, altura normalizada, o features derivadas.
  • Fine-tuning: Pre-entrenar en sintéticos y fine-tune con pocos datos reales.

5. Conclusiones

  • La arquitectura PointMLP-Elite con 192K parámetros es capaz de clasificar 3 categorías arbóreas con 98.7% de accuracy en datos sintéticos y 53.2% en datos reales.
  • El domain gap es severo: el modelo entrenado en sintéticos cae a 29.9% en datos reales (−68.8pp), apenas por encima del azar.
  • Los datos sintéticos en su forma actual no son un sustituto viable para datos LiDAR reales en esta tarea de clasificación.
  • Ambos modelos muestran sesgos de clase opuestos al cruzar dominios, indicando que las representaciones geométricas aprendidas son específicas de cada dominio.
  • Se recomienda explorar domain adaptation y mejorar la calidad de los datos sintéticos antes de considerarlos útiles para entrenamiento.
  • La clase OTHER es consistentemente la más difícil, sugiriendo que una clasificación binaria (PINE vs no-PINE) o con clases más homogéneas podría mejorar resultados.
